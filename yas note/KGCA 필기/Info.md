# Info

* ## Library
    여러 프로그램에서 자주 사용하는 함수와 데이터들을 실행이 가능한 바이너리 형태로 묶어놓은 것

  * 정적 라이브러리 (.lib)  
  : 실행파일 안에 라이브러리가 하나로 묶인다.  
    실행파일의 크기는 커지지만 실행파일만 있으면 어느 곳에서나 실행가능하다.  
    프로그램 실행 후 처리시간이 빠르다.

  * 동적 라이브러리 (.dll)  
  : 실행파일 안에 포함되지 않고, 프로그램 실행 시에 참조된다.  
  실행파일의 크기가 작으며 물리 메모리에 한번 올라간 dll은 둘 이상의 프로세스가  
  각각 자신의 가상 메모리에 매핑해서 공유한는 구조를 지니기 때문에 메모리 사용 효율이 좋다.  
  런타임 중 필요할 때 외부 dll 파일에서 참조한다.  
  dll을 만들 때, lib 파일도 같이 만들어지는데 이 lib파일은 dll이 제공하는 함수 정보를 가지고 있다.  
  즉 생성되는 lib는 링크 dll은 실행이다.  
  dll은 반드시 실행 파일과 같은 폴더에 있어야 한다.
  **dll은 코드만 가져다 쓰고 exe에는 포함되지 않는다.**
